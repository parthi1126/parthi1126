### 📋 EpicalExam — Cloud-Native Employee Assessment Platform
> **`Python` `Flask` `Google Sheets API` `GCP Cloud Run` `Gunicorn` `Bootstrap`**

[![Live Demo](https://img.shields.io/badge/🌐_Live_Demo-GCP_Cloud_Run-4285F4?style=for-the-badge)](https://epical-exam1-856878813474.asia-south1.run.app/)
[![Repo](https://img.shields.io/badge/📦_Source-GitHub-181717?style=for-the-badge&logo=github)](https://github.com/parthi1126/EpicalExam)
![Commits](https://img.shields.io/badge/Commits-41-FF6B6B?style=for-the-badge)
![Stars](https://img.shields.io/badge/Stars-⭐_1-FFD700?style=for-the-badge)

**Real-world internal exam portal deployed on Google Cloud Platform for Epical Layouts.** A full-stack serverless application that replaces a traditional SQL database with Google Sheets as a live, zero-cost data backend.

```
🏗️  Architecture:  Flask Backend  →  gspread Client  →  Google Sheets (live DB)
☁️  Deployment:    Docker Container  →  GCP Cloud Run (asia-south1, Mumbai)
🔐  Security:      Session tokens  +  IsActive concurrent-login guard
⚡  Resilience:    Exponential backoff decorator  →  handles API quota (HTTP 429)
📊  Admin Panel:   Real-time leaderboard  +  Excel export  +  live config edits
🧪  Exam Engine:   MCQ (single + multi-choice)  +  server-side auto-grading
```

**Key Engineering Decisions:**
- Google Sheets as a zero-setup live database — no SQL, no migrations, instant admin edits
- Custom `@retry_on_quota_exceeded` decorator with exponential backoff + jitter
- Concurrent-login prevention via `IsActive` flag (prevents cheating / session abuse)
- Server-side answer re-grading on submission (never trusts client-side data)
- Random stagger delay `(0.5–2.0s)` on simultaneous submissions to prevent API collisions
- Multi-variant exam support: `Questions_TEST{N}` worksheet naming convention

### 🤖 RAG Application — Retrieval-Augmented Generation Pipeline
> **`Python` `LangChain` `ChromaDB / FAISS` `OpenAI` `Vector Embeddings`**

[![Repo](https://img.shields.io/badge/📦_Source-GitHub-181717?style=for-the-badge&logo=github)](https://github.com/parthi1126/rag_application)
![Lang](https://img.shields.io/badge/Python-100%25-3776AB?style=for-the-badge&logo=python&logoColor=white)

**A from-scratch RAG pipeline** that grounds LLM responses in a private document corpus — the backbone of any production AI assistant.

```
📄  Document Ingestion  →  Text Chunking  →  Embedding Generation
🔍  Vector Store (ChromaDB/FAISS)  →  Semantic Retrieval  →  Context Injection
🤖  LLM Response  →  Grounded, citation-backed, hallucination-reduced output
```

**Core Concepts Implemented:**
- Document chunking strategies (recursive character splitter)
- Embedding model integration (OpenAI / HuggingFace sentence transformers)
- Vector similarity search for context retrieval
- Prompt engineering for grounded QA chains
